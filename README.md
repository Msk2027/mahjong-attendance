🀄 麻雀出欠ボード（Mahjong Attendance Board）

麻雀の開催日程調整・出欠管理をシンプルに行うためのWebアプリです。
友人同士の麻雀会やサークル活動など、少人数グループでの利用を想定しています。

URL：
👉 mahjong-attendance.vercel.app

⸻

📌 主な特徴

✅ ルーム単位での出欠管理
	<br>•	ルームを作成し、招待コードやURLでメンバーを招待
	<br>•	メンバーは複数人で参加可能

📅 日程候補の追加・出欠調整
	<br>•	開催候補日を追加
	<br>•	各メンバーが ◯ / △ / × で出欠を入力
	<br>•	ゲスト参加者も候補日に紐づけて追加可能
	<br>•	ゲストは確定参加として自動集計

🎯 開催確定フロー
	<br>•	最低参加人数に到達した候補日に対してオーナーのみ「開催確定」操作が可能
	<br>•	開催確定後はイベントとして管理

🧑‍🤝‍🧑 開催詳細ページ
	<br>•	開催日の参加者一覧（メンバー・ゲスト）
	<br>•	各参加者が 後から参加 / 不参加を変更可能
	<br>•	オーナーは他メンバーの参加状態も調整可能
	<br>•	開始時刻・メモ（集合場所など）の共有

👤 ゲスト参加対応　→バグにより削除中
	<br>•	アカウントを持たないゲストを簡単に追加
	<br>•	イベント単位で参加者として管理

🔐 アカウント設計
	<br>•	匿名アカウント（体験版）での利用が可能
	<br>•	後からメールアドレス・パスワードを設定して
アカウントを引き継ぐことも可能

⸻

🛠 技術スタック

フロントエンド
	<br>•	Next.js (App Router)
	<br>•	TypeScript
	<br>•	CSS（カスタムユーティリティクラス）

バックエンド / BaaS
	<br>•	Supabase
	<br>•	Authentication（匿名ログイン対応）
	<br>•	PostgreSQL
	<br>•	Row Level Security（RLS）
	<br>•	RPC（PL/pgSQL 関数）

⸻

🗂 データ設計の概要
	<br>•	rooms：ルーム情報
	<br>•	room_members：ルーム所属メンバー
	<br>•	schedule_candidates：開催候補日
	<br>•	rsvps：候補日への出欠
	<br>•	room_guests：候補日ごとのゲスト
	<br>•	events：開催確定イベント
	<br>•	event_participants：開催日の参加者スナップショット

※ 開催確定時に 候補日時点の参加者情報をイベントにコピーする設計

⸻

🔒 権限設計（RLS）
	<br>•	ルームメンバーのみ閲覧・操作可能
	<br>•	オーナー権限
	<br>•	開催確定
	<br>•	他メンバーの参加状態変更
	<br>•	メンバーの追加 / 削除
	<br>•	一般メンバー
	<br>•	自分の出欠変更
	<br>•	ゲストの追加 / 削除

⸻

💡 工夫した点（アピールポイント）
	<br>•	匿名ログイン対応による参加ハードルの低減
	<br>•	「日程候補 → 開催確定 → イベント管理」という状態遷移を明確に分離した設計
	<br>•	RLS + RPC による 安全な権限管理
	<br>•	スマホ利用を前提としたUI調整
	<br>•	後から機能拡張（結果入力・成績管理）が可能な構造
